### 一、 核心概念：后进先出 (LIFO)

栈是一种**线性数据结构**，它的主要特征是**后进先出**（Last In, First Out，简称 LIFO）。

你可以把它想象成食堂里叠放的餐盘，或者一摞书：

- **入栈（Push）：** 当你放一个新盘子时，只能放在最上面。
    
- **出栈（Pop）：** 当你需要拿一个盘子时，也只能从最上面拿走。
    
- **栈顶（Top）：** 唯一允许进行插入和删除操作的一端。
    
- **栈底（Bottom）：** 封闭的一端，最先放入的元素会一直压在栈底，直到它上面的所有元素都被移走。


### 二、 栈的底层实现

栈通常有两种常见的底层实现方式，它们在执行 `Push` 和 `Pop` 操作时的时间复杂度都是 $O(1)$（常数时间）：

**1. 基于数组实现 (顺序栈)**

- **原理：** 分配一块连续的内存空间（数组），并使用一个变量（通常叫 `top`）来记录栈顶元素在数组中的索引。
    
- **优点：** 内存连续，读写速度快，且节省了存储指针的额外空间。
    
- **缺点：** 静态数组大小固定，如果元素超过容量会导致“栈溢出（Stack Overflow）”；动态数组在扩容时会有数据搬运的性能损耗。


**2. 基于链表实现 (链式栈)**

- **原理：** 使用单链表，将链表的头部作为栈顶。每次 `Push` 相当于在链表头部插入节点（头插法），`Pop` 相当于删除头节点。
    
- **优点：** 容量理论上没有限制（只要内存足够），不需要考虑扩容问题。
    
- **缺点：** 每次操作都需要动态分配或释放节点内存，且每个节点需要额外存储指向下一个节点的指针。


### 三、 出栈序列的方案数（卡特兰数）

这是关于栈的最经典数学性质之一。

**问题：** 假设有 $n$ 个元素按照 $1, 2, 3, \dots, n$ 的顺序依次进栈（中途可以随时出栈），那么一共可能产生多少种不同的出栈序列？

**结论：** 方案数等于第 $n$ 个**卡特兰数（Catalan Number）**。

其数学公式为：

$$H_n = \frac{1}{n+1} \binom{2n}{n} = \frac{(2n)!}{(n+1)!n!}$$

**为什么是卡特兰数？**

我们可以把操作抽象一下：

1. 每一次“入栈”记作 $+1$。
    
2. 每一次“出栈”记作 $-1$。


要使序列合法，必须满足一个硬性条件：**在任何时刻，已经出栈的元素数量绝不能超过已经入栈的元素数量。** 也就是说，任何前缀和必须 $\ge 0$。最终，$n$ 次入栈和 $n$ 次出栈的总和为 $0$。

这在组合数学中与以下经典问题是完全等价的：

- $n$ 对括号的合法匹配方案数。
    
- 在一个 $n \times n$ 的网格中，从左下角走到右上角，且不越过对角线的路径数。


> **举个例子 ($n=3$)：**
> 
> 元素 1, 2, 3 依次入栈，合法的出栈序列有 5 种（根据公式 $H_3 = \frac{1}{4} \binom{6}{3} = \frac{20}{4} = 5$）：
> 
> 1. 1进 1出, 2进 2出, 3进 3出 $\rightarrow$ 序列: `1, 2, 3`
>     
> 2. 1进 1出, 2进 3进 3出 2出 $\rightarrow$ 序列: `1, 3, 2`
>     
> 3. 1进 2进 2出 1出, 3进 3出 $\rightarrow$ 序列: `2, 1, 3`
>     
> 4. 1进 2进 2出 3进 3出 1出 $\rightarrow$ 序列: `2, 3, 1`
>     
> 5. 1进 2进 3进 3出 2出 1出 $\rightarrow$ 序列: `3, 2, 1`
>     
>     _(注意：`3, 1, 2` 是不可能的，因为 3 先出栈说明 1 和 2 都在栈里，且 2 压在 1 上面，出栈顺序只能是 2 然后 1。)_


[栈模板](https://www.luogu.com.cn/problem/B3614)

纯C语言实现栈：
```cpp
#include <bits/stdc++.h>
using namespace std;
typedef unsigned long long ull;
ull stacks[1000005];
int top = 0;
void push(ull x)
{
    stacks[top++] = x;
}
void pop()
{
    top--;
}
bool exits_top()
{
    return top > 0;
}
ull query()
{
    return stacks[top - 1];
}
int size()
{
    return top;
}
void solve()
{
    top = 0;
    int n;
    cin >> n;
    string s;
    ull x;
    while (n--)
    {
        cin >> s;
        if (s == "push")
        {
            cin >> x;
            push(x);
        }
        else if (s == "pop")
        {
            if (exits_top())
            {
                pop();
            }
            else
            {
                cout << "Empty" << "\n";
            }
        }
        else if (s == "query")
        {
            if (exits_top())
            {
                cout << query() << "\n";
            }
            else
            {
                cout << "Anguei!" << "\n";
            }
        }
        else if (s == "size")
        {
            cout << size() << "\n";
        }
    }
}
int main()
{
    int t;
    cin >> t;
    while (t--)
        solve();
}
```

使用C++中的STL实现：
```cpp
#include <bits/stdc++.h>
using namespace std;
typedef unsigned long long ull;
void solve()
{
    int n;
    cin >> n;
    stack<ull> stack;
    string s;
    ull x;
    while (n--)
    {
        cin >> s;
        if (s == "push")
        {
            cin >> x;
            stack.push(x);
        }
        else if (s == "pop")
        {
            if (!stack.empty())
            {
                stack.pop();
            }
            else
            {
                cout << "Empty" << "\n";
            }
        }
        else if (s == "query")
        {
            if (!stack.empty())
            {
                cout << stack.top() << "\n";
            }
            else
            {
                cout << "Anguei!" << "\n";
            }
        }
        else if (s == "size")
        {
            cout << stack.size() << "\n";
        }
    }
}
int main()
{
    int t;
    cin >> t;
    while (t--)
        solve();
}
```

# 四、一些关于栈的理解
在图灵完备的体系下，有一条著名的铁律：**任何递归程序都可以用显式的栈（配合循环）改写为非递归（迭代）程序；反之，任何依赖栈的循环程序，也都可以改写为递归程序。** 它们本质上是**同一种后进先出（LIFO）逻辑结构，在不同维度的实现**。
#### 1. 本质上的同源性

递归和栈之所以能相互转化，是因为**递归的底层物理实现本身就是一个栈（系统调用栈）**。

- **递归**是“隐式”地使用了操作系统提供的栈空间来保存每一次函数调用的状态（局部变量、参数、返回地址）。
    
- 显式栈（迭代）则是程序员在堆内存中“手动”模拟了这个过程，自己定义数据结构来保存需要的状态。

#### 2. 转化方向一：递归 $\rightarrow$ 栈（将递归展开为迭代）

当我们面临深度极大的递归，为了防止栈溢出（Stack Overflow）或追求极致运行效率时，需要将递归转化为显式栈。

- **转化思路：** 将递归函数中需要传递的参数、局部变量以及当前执行到的“阶段”打包成一个节点，手动压入自己创建的栈中。用 `while (栈不为空)` 循环代替函数的自我调用。
    
- **核心代价：** 代码逻辑变复杂，丧失了递归的简洁性与可读性；但换取了极大的空间安全性。

#### 3. 转化方向二：栈 $\rightarrow$ 递归（用递归替代显式栈）

当我们在写诸如深度优先搜索（DFS）、树的遍历、或者解析嵌套结构时，如果发现自己正在手动维护一个栈，通常可以直接用递归来重构。

- **转化思路：** 把原先“压栈”的动作变成一次“函数调用”，把“出栈”的动作变成函数的 `return`。
    
- **核心代价：** 引入了函数调用的额外开销（时间上的上下文切换，空间上的系统栈帧消耗）；但换取了极其优雅、高度契合人类数学逻辑的代码。
